1. QWAT / Module TEKSI eau potable¶

Le module TEKSI eau potable (nom de projet : QWAT) est une sélection d’outils et une implémentation de base de données qui vous permet de :

  • gérer et cartographier votre réseau de distribution d’eau avec tous ses composants: conduites, vannes, pompes, réservoirs, etc.

  • récolter les attributs des objets du réseau tels que diamètres, matériaux, profondeurs, fuites, dates d’installation, etc.

  • générer des plans, cibler les interventions à prévoir et extraire des statistiques comme la valeur du réseau, la longueur totale des conduites, etc.

  • exporter des géodonnées dans les modèles de données standards suisses.

TEKSI eau potable est un module open source complet basé sur un modèle de données PostgreSQL - PostGIS inspiré de la norme suisse SIA405. Les données sont accessibles et éditables à l’aide d’un projet QGIS dédié. Le projet est maintenu par la communauté TEKSI.

1.1. Principales fonctionnalités¶

1.1.1. Objets du réseaux¶

Le modèle de données contient tous les objets d’un réseau de distribution d’eau et leurs attributs :

  • Conduites

  • Réservoirs

  • Vannes

  • Chambres

  • Couvercles

  • Stations de traitement

  • Pompes

  • Installations de contôle de pression

  • Sources

  • Installations de mesures

  • Hydrantes

  • Abonnés

  • Points de prélèvements

  • Fuites

L’altitude des objets est enregistrée directement dans la géométrie. Celle-ci peut être modifiée avec les outils natifs de QGIS.

Le modèle de données est quasi compatible avec la SIA405. Le modèle est complet et fourni tous les objets et attributs courants. Cependant, il est possible d’adapter les objets et attributs à ses propres besoins ou de proposer des modifications à la communauté.

1.1.2. Consultation et édition¶

Un projet QGIS pré-configuré est fourni. Les objets du réseau sont représentés avec une symbologie standard.

L’édition peut être faite directement depuis QGIS, chaque objet possède un formulaire dédié qui facilite l’édition des attributs.

Documentation sur l’édition

1.1.3. Impression¶

Des mises en page pré-configurées sont disponibles dans le projet QGIS. La géométries des conduites et les étiquettes sont adaptées à l’échelle d’impression.

Documentation sur l’impression

1.1.4. Export¶

Une librairie Python est disponible pour exporter le réseau dans le modèle de données SIA405 en interlis.

Des extensions sont développées pour d’autres standards:

  • SIRE (Canton de Vaud)

  • AquaFri (Canton de Fribourg)

  • RAEPA - COVADIS (France)

1.1.5. History viewer¶

Le plugin PG History Viewer peut être configuée pour QWAT. L’extension PostgreSQL permet de tracker les changements effectués sur les données. L’interface QGIS permet de sélectionner et annuler des changements.
